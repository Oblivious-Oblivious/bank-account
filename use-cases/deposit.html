Deposit
Data:
  • <Cash-amount>
Primary Course:
  1. User issues "Deposit Amount" command with above data.
  2. System validates that cash amount is an acceptable value.
  3. System adds amount to the users account and determines transaction-id.
  4. System delivers transaction-id to user.
Large Amount Error (2)
  1. System requests that a big enough amount shall be deposited in-person.
  2. System terminates "Deposit".
