Send
Data:
  • <Recipient-hash-id>
  • <Transfer-amount>
  • <Fee-amount>
Primary Course:
  1. User issues the "Send" command with above data.
  2. System validates all data.
  3. System calculates the transaction priority according to the fee provided.
  4. System notifies the receiver and performs "Deposit" on his account.
  5. System performs "Withdraw" and provides a transaction-id to the sender.
Hash ID Error (2)
  1. System identifies a non existent hash id.
  2. System repeats "Send".
Amount Error (2)
  1. System determines that the user does not have enough funds.
  2. System repeats "Send".
